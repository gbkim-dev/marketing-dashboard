# Marketing Dashboard Redesign — Design Spec

**Date:** 2026-05-15
**Team:** 퍼포먼스 마케팅
**Status:** Approved (브레인스토밍 단계 종료)

## 목적

매일 아침 처음 열어보는 화면으로, **"오늘 따져봐야 할 캠페인이 뭐인가"** 를 30초 안에 파악하는 대시보드. 운영자가 매일 단순 반복으로 KPI 표를 훑던 작업을 자동화 + 우선순위화한다.

## 핵심 원칙

1. **이상 신호 우선** — 평균·합계보다 변화·이상치 강조
2. **계층 드릴다운** — 캠페인 → 그룹 → 소재 (액션 단위에서 진단 단위로)
3. **듀얼 베이스라인** — 전일(D-2) 메인, 전주 동요일(D-8) 보조 (요일 효과 보정)
4. **데이터 그대로** — 데이터 없으면 솔직히 "—" 표시, 가짜 채워넣지 않음

## 아키텍처

```
marketing-dashboard/
├── app.py                       ← Streamlit entry (4-tab router)
├── lib/
│   ├── loader.py                ← CSV 스캔·조인 (기존 유지)
│   ├── anomaly.py               ← [NEW] 베이스라인 비교·심각도 분류
│   └── kpi.py                   ← [NEW] KPI 공식·집계
├── components/
│   ├── tab_anomaly.py           ← 🚨 이상 신호 (메인 탭)
│   ├── tab_summary.py           ← 📊 요약
│   ├── tab_trend.py             ← 📈 추세
│   └── tab_breakdown.py         ← 🧩 분해
├── data/
│   └── YYYY-MM-DD/
│       ├── channel.csv
│       └── appsflyer.csv
└── docs/superpowers/specs/      ← 이 문서
```

**경계 원칙:** 각 `components/tab_*.py` 는 자기 데이터·렌더링만 책임. 공유 로직(베이스라인 계산, KPI 공식)은 `lib/` 에. 한 탭 코드 읽으면 그 탭이 뭐 하는지 명확.

## 데이터 흐름

```
CSV files (data/YYYY-MM-DD/)
   ↓
loader.load_channel() + load_appsflyer()  → DataFrame
   ↓
merge_datasets() — 일·캠페인·그룹·소재 4-key 조인
   ↓
@st.cache_data(ttl=60) — 폴더 변경 시 60초 내 자동 반영
   ↓
anomaly.detect(df, baseline="D-2" or "D-8") → severity-tagged DataFrame
   ↓
각 탭 컴포넌트가 필터된 데이터로 렌더링
```

## 탭별 명세

### 🚨 TAB 1 — 이상 신호 (메인)

**입력**: 전체 merged DataFrame (최근 8일 이상 권장)
**필터**: 사이드바 (날짜·채널·캠페인·심각도)

**상단 요약 칩**
- 🔴 심각 N건 (CPA +30%↑ 또는 ROAS -30%↓)
- 🟡 주의 N건 (CPA +15~30% 또는 ROAS -15~-30%)
- 🟢 정상 N건

**메인 표** (기준일=D-1, 비교=D-2 / D-8)
| 컬럼 | 내용 |
|------|------|
| 캠페인 | `▼/▶` + 이름 |
| CPA D-1 | 어제 절대값 (원) |
| vs D-2 | 전일 대비 % 변화 (▲▼ 색상) |
| vs D-8 | 전주 동요일 대비 % 변화 |
| 비용 영향 | D-1 비용 (정렬용 — 큰 손실 먼저) |

**행 색상**
- 🔴 심각: `border-left: 3px solid #dc2626` + 빨간 배경
- 🟡 주의: 노란 배경
- 🟢 정상: 기본 배경 (접힌 상태로)

**드릴다운 동작**
- `▶` 클릭 → 같은 행에서 펼쳐서 그룹 → 소재 순으로 표시 (들여쓰기로 계층 시각화)
- Streamlit 구현: `st.session_state[expanded_campaigns]` set 으로 토글 제어
- 펼친 행은 다음 동일 키로 자동 보존

**정렬**
- 1차: 심각도 (심각 → 주의 → 정상)
- 2차: 비용 영향 큰 순 (예산 손실 큰 것부터)

**임계치 (사이드바에서 조정 가능)**
- 기본: CPA ±15% / ±30%, ROAS ±15% / ±30%
- 추가 지표: CTR, CVR (선택)

### 📊 TAB 2 — 요약

**목적**: 전일 종합 KPI 1페이지 + 광고주 보고용 캡처

**구성**
- KPI 카드 8개: 비용 · 노출 · 클릭 · CTR · CPC · CPA · ROAS · 회원가입
- 채널별 비용 분포 (도넛 차트)
- 어트리뷰션 갭 콜아웃 (매체 vs AF ±5% 초과 시)
- 전일 대비 모든 KPI delta 표기 (▲▼)

### 📈 TAB 3 — 추세

**목적**: 최근 7일/30일 시계열

**구성**
- 메트릭 토글: CPA / ROAS / CTR / CVR (one at a time)
- 기간 토글: 7일 / 14일 / 30일
- 라인 색상: 채널별 (구글·메타·네이버)
- 호버: 일별 정확한 값 + 캠페인 수

### 🧩 TAB 4 — 분해

**목적**: 임의 차원으로 슬라이싱 + CSV 다운로드

**구성**
- 차원 선택 멀티셀렉트: 채널 / 캠페인 / 캠페인목적 / 그룹 / 소재 (조합 가능)
- 정렬 키: 비용 / ROAS / CPA / 회원가입
- Top N 슬라이더: 5 / 10 / 20 / 전체
- 표 + 막대 차트 분기
- "⬇️ CSV 다운로드" 버튼

## 에러 처리 / 엣지 케이스

| 상황 | 동작 |
|------|------|
| `data/` 폴더 비어있음 | 안내문: "data/YYYY-MM-DD/ 폴더 만들고 channel.csv·appsflyer.csv 추가하세요" + README 링크 |
| 단일 일자만 존재 | 이상 신호 탭: "베이스라인 비교 데이터 부족 — 최소 2일치 필요" 안내. 다른 탭은 그대로 작동 |
| D-8 데이터 없음 (D-2는 있음) | vs D-8 컬럼에 "—" 표시, vs D-2만으로 심각도 판정 |
| 캠페인이 어제만 있고 D-2 없음 (신규 캠페인) | 라벨 "신규 캠페인 — 비교 불가" + 정상 영역에 배치 |
| 어트리뷰션 갭 25%+ | 🚨 자동 콜아웃 (매 탭 상단) |
| CSV 파싱 실패 | 해당 파일 스킵 + 사이드바에 경고 ("3개 중 1개 파일 읽기 실패: ...") |

## 테스트 시나리오

| 케이스 | 검증 포인트 |
|--------|-----------|
| 단일 일자 (현 상태) | 이상 신호 탭 — 데이터 부족 안내 표시. 다른 탭 정상 작동 |
| 8일치 합성 데이터 + 인위적 anomaly 1개 | 이상 신호 탭에 빨간 카드 1개 표시. 펼치면 그룹·소재 표시 |
| D-1 데이터만 있고 D-2, D-8 없음 | "비교 불가" 안내 |
| 어트리뷰션 갭 30% | 모든 탭 상단에 콜아웃 |

## 구현 우선순위

1. **데이터 보강**: 합성 데이터 생성 스크립트 (8일치, 1개 anomaly injected)
2. **lib/anomaly.py**: 베이스라인 비교·심각도 분류 로직
3. **components/tab_anomaly.py**: 핵심 탭 (인라인 드릴다운 포함)
4. **app.py**: 4탭 라우터
5. **components/tab_summary.py · tab_trend.py · tab_breakdown.py**: 보조 탭 (기존 코드 분할 재활용)
6. **검증**: 합성 데이터로 모든 시나리오 통과 확인

## 의도적 비포함 (YAGNI)

- 자동 슬랙·이메일 알람 — `Claude는 절대 자동 발송 안 함` 원칙 (CLAUDE.md)
- 예산 자동 조정 추천 — 사람 검수 필수
- 광고주별 멀티 워크스페이스 — 현재 단일 광고주, 필요 시 후속 spec
- 모바일 반응형 — 데스크톱 우선
- 로그인·권한 — 로컬 단일 사용자 가정

## CLAUDE.md 컨벤션 활용

- 캠페인 prefix `GGL_/META_/NVR_` → 채널 자동 매핑 (이미 `loader.py` 에 있음)
- 소재 타입 prefix `VID_/IMG_/CRS_/TXT_` → 분해 탭에서 그룹화 옵션
- 그룹명 (논타겟·유사타겟·VIP·윈백·리마케팅) → 정렬·필터 옵션
- 어트리뷰션 정책: AF 우선, 매체 보조 (CLAUDE.md 정책 따름)
